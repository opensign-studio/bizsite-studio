# Outreach Templates

Slots: `{business}`, `{owner_or_team}` (use "Hi there" / "Hi {business} team" if no name is public), `{site_url}`, `{specific_compliment}` (from the research fact sheet — public, verifiable), `{history_hook}` (their founding year / signature detail), `{domain_note}` (if they own a domain), `{user_name}`, `{user_email}`.

`[ASK USER: connection]` — the user's true relationship to the business. Options to offer the user: "I'm a customer", "I live in the neighborhood", "I'm a local developer who admires the business". NEVER auto-fill this slot. If the user has no connection, the third option is honest and still warm.

## 1 — First touch

Subject options (pick per business):
- `I built {business} a website — take a look`
- `A website for {business} (already live)`
- `{history_hook} deserves a website` (e.g., "94 years on Monroe Drive deserves a website")

```
{owner_or_team},

[ASK USER: connection — one sentence, true]. {specific_compliment — one
sentence about something real: their reviews, their signage, their
reputation}. Which is why it surprised me that {business} doesn't have a
website of its own / is still running on a website from another era.

So I went ahead and built one: {site_url}

It's already live — your real hours, your services, your photos,
{history_hook if strong}. Nothing's required of you; I built it because I
think {business} deserves it.

If you like it, it's yours. {domain_note — e.g., "You already own
sigsamuels.com, so pointing it at the new site takes about ten minutes."}
I'd love to {single CTA: "swing by the shop" / "hop on a ten-minute call"} —
whatever's easiest.

{user_name}
{user_email}
```

## 2 — Follow-up (once, 5–7 days, only if no reply)

Subject: `Re:` the original.

```
{owner_or_team},

Just floating this back up — the site I built for {business} is still live
here: {site_url}

No pressure at all. If it's not something you're interested in, no hard
feelings — I'll keep being a fan of the shop either way. If it is, a
ten-minute call is all we'd need.

{user_name}
```

## 3 — After a positive reply (three lanes — confirm terms with the user first)

Never invent prices. Ask the user for their real numbers before drafting any of these.

**(a) Walkthrough**: propose 2–3 concrete times; offer in-person or call; promise "15 minutes, I'll show you the site on your phone."

**(b) Straight purchase/handoff**: what they get — the site, the code (GitHub transfer), the hosting setup (Vercel project transfer or redeploy under their account), their domain connected, and a walkthrough. One flat figure `[USER'S PRICE]`. Note transfers are easiest done together on a short call.

**(c) Maintenance arrangement**: user keeps hosting/repo; owner gets the site live on their domain plus updates (hours, photos, seasonal notes) within `[USER'S SLA]` for `[USER'S MONTHLY FEE]`/mo, cancel anytime, and if they cancel the site and domain setup are still theirs. This recurring lane is usually the better mutual deal — say so plainly.

## Sending checklist (from SKILL.md, abbreviated)

Recipient is a published address → draft via Gmail connector → show user → explicit yes → send via Gmail web (drafts list → click row → expand minimized compose bar → Cmd+Return → confirm "Message sent" toast) → mark "Contacted" in the pipeline artifact → offer the 5–7 day follow-up reminder.
